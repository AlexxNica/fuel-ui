This simple configuration is for you if you want to give OpenStack a test drive. You will get a fully functional private cloud, with all OpenStack components installed on one node. The node will also act as a compute node.

In this configuration OpenStack controller is deployed separately from compute and nodes. The mode assumes presence of 1 controller node and 1 or more compute/storage nodes. You can add more nodes as you go and scale your cloud.

This configuration requires multiple OpenStack controllers (3+) and provides high availability for all OpenStack components, including MySQL/Galera, RabbitMQ, Quantum, Cinder, as well as OpenStack API services. Select this configuration if you want to build a production-grade OpenStack cloud with 6 nodes or more.

In this deployment type your installation will have only computes to run VMs on them. There will be no storage (Swift) nodes.

Choose this deployment type if you don't need to run virtual machines and need only storage (Swift) nodes.

Choose this deployment type if you need both compute and storage nodes, to run VMs and store objects into Swift.
